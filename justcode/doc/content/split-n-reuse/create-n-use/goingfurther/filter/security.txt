==================================
Attention aux filtres maléfiques !
==================================

En interne, un filtre est un ¨prog ¨python. Il est donc risqué d'autoriser aveuglément l'usage de filtres "maison". C'est pour cette raison que l'utilisateur doit explictement autoriser l'usage de filtres proposés par un patron via le bloc peuf``main::...`` comme suit où l'on indique le nom du patron autorisé à ajouter un nouveau filtre.

peuf::
    ---
    path = ¨exafilter/params.peuf
    ---


La valeur de peuf::``filters`` peut aussi être de l'un des deux types suivants.

    1) Plusieurs patrons peuvent être autorisés comme dans peuf::``filters = patron_1 ; patron_2 ; patron_3``.

    1) peuf::``filters = all`` permet d'autoriser tous les patrons utilisés à fournir des filtres.

    1) Par défaut, peuf::``filters = no`` interdit tout filtre proposé par un patron.


tip::
    Il est conseillé de ne coder que de petits filtres facilement analysables par un habitué de ¨python.
    De plus, il faut cibler les importations en évitant par exemple l'usage de python::``from os import *``.
